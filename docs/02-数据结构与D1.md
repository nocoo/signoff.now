## 2. 设计原则

| 原则 | 决策 |
|:-----|:-----|
| 单实例 | 无 tenant_id |
| 主键 | TEXT UUID |
| 时间 | unix **秒** UTC；自然日 `day_key` + **config_version** |
| 软删除 | `archived_at`；活跃行 UNIQUE 用 partial index；**不**硬删有历史的 Developer/Repo |
| 历史 FK | Activity/Score → Developer/Repo 使用 **ON DELETE RESTRICT** |
| 幂等 | `activities.external_ref` 全局 UNIQUE；键基于 **稳定 id**（§5） |
| 配置失效 | `pipeline_config_version` + `scores_stale`；改 timezone/weights 必须 bump 并标 stale |
| 聚合去重 | 「同人同 WI 同日一次」在 **Score 计算层**，不在 external_ref 用 day_key 压扁多人 |
| JSON | `settings.value`、`scores.breakdown_json`、`activities.meta_json` 使用 `json_valid()` |
| 读写边界 | Activity/Score 仅管线写 |

## 3. 配置版本与强制重算（P1）

### 3.1 Settings 键

| key | 类型 | 说明 |
|:----|:-----|:-----|
| `timezone` | string | 默认 `Asia/Shanghai` |
| `email_suffixes` | string[] | 默认 `["microsoft.com"]` |
| `activity_weights` | object | 见 01 §6.3 |
| `pipeline_config_version` | number | **单调递增**；默认 `1` |
| `scores_stale` | boolean | 配置变更后 `true`，全量重算成功后 `false` |
| `scores_stale_reason` | string \| null | UI 提示 |

`value` 一律 JSON 文本，且 `CHECK (json_valid(value))`。

### 3.2 写入语义

| 操作 | 必须 |
|:-----|:-----|
| Web 修改 `timezone` 或 `activity_weights` | `pipeline_config_version += 1`；`scores_stale = true`；写入 reason |
| 仅改 `email_suffixes` | **不**强制 bump（影响匹配，不改 day_key/权重）；未映射列表可能变化 |
| 管线 ingest 单条 Activity | 写入当前 `pipeline_config_version` 到 `activities.config_version`；按该版本时区算 `day_key` |
| 管线计算 Score | `scores.config_version = 当前版本`；仅聚合 `activities.config_version = 当前版本` 的行（或重算任务统一重写 day_key） |
| 全量重算任务 | 对全部 Activity：用**当前** timezone 重写 `day_key` + `config_version`；按 developer×day_key 重建 Score；成功后 `scores_stale=false` |

### 3.3 UI

- 若 `scores_stale === true`：热力图/对比展示 **stale 横幅**，禁止当作最终考核结果。  
- 提供「触发全量重算」（本机 skill/script 或受控 API 调起本机任务——鉴权见 06）。

### 3.4 行级字段

| 表 | 列 | 说明 |
|:---|:---|:-----|
| `activities` | `config_version` | 生成该 `day_key` 时的配置版本 |
| `scores` | `config_version` | 计算该分时的配置版本 |

查询「当前有效积分」：`WHERE config_version = (SELECT json pipeline_config_version) AND scores_stale = false`（应用层组合判断）。

## 5. `external_ref` 稳定约定（P1）

**禁止**把可变的 org 显示名 / project 名 / repo 名 / 邮箱作为永久幂等身份。  
**禁止** `wi.updated:{day_key}` 作为唯一键（同一天多人对同一 WI 会冲突）。

### 5.1 稳定标识来源（ADO）

| 实体 | 稳定 id |
|:-----|:--------|
| Repository | `repos.external_id` = ADO **repository GUID**（绑定时写入） |
| Project | ADO **project GUID**（可存 meta 或拼进 ref；名称列仅展示） |
| Pull Request | `pullRequestId`（在 repo 内稳定）+ repo GUID |
| Identity（vote 主体） | ADO **identity id** 或 **descriptor**（事件载荷中的 id，不是 email） |
| Work Item | `System.Id` + project GUID |
| WI 修订 | **revision id** / update id（updates API） |
| PR thread/comment | **thread id** + **comment id**（threads API） |

名称列（`repos.org/project/name`）仍保留供展示与 az CLI 调用参数；**幂等键只用 GUID**。

### 5.2 推荐 ref 模板

| type | external_ref |
|:-----|:-------------|
| `pr.created` | `ado:pr:{repoGuid}:{prId}:created` |
| `pr.merged` | `ado:pr:{repoGuid}:{prId}:merged` |
| `pr.closed` | `ado:pr:{repoGuid}:{prId}:closed` |
| `pr.vote` | `ado:pr:{repoGuid}:{prId}:vote:{identityId}:{threadId}:{commentId}` |
| `pr.active` | `ado:pr:{repoGuid}:{prId}:iter:{iterationId}` |
| `wi.created` | `ado:wi:{projectGuid}:{wiId}:created` |
| `wi.closed` | `ado:wi:{projectGuid}:{wiId}:closed` |
| `wi.updated` | `ado:wi:{projectGuid}:{wiId}:rev:{revisionId}` |

### 5.3 Score 层去重（非 ref 层）

- **同人同 WI 同日多次 `wi.updated`**：保留多条 Activity（每 revision 一条），Score 聚合时对该 type **按日计一次权重**（或取 max/sum 策略在 03 定，默认「按日计一次」）。  
- **同人同 PR 同日多个 active iteration**：Score 层可按日折叠 `pr.active`。  
- 终态 PR（merged/closed）与 created 的优先级仍在 03 算法中处理。

## 6. 表定义（现行 = 0001 + 0002）

### 6.1 `settings`

| 列 | 约束 |
|:---|:-----|
| `key` | PK |
| `value` | NOT NULL，`CHECK (json_valid(value))` |
| `updated_at` | unix 秒 |

键见 §3.1。

### 6.2 `developers` / `teams` / `tags`

| 列 | 说明 |
|:---|:-----|
| `id` | UUID |
| `name` | 显示名 / 名称 |
| `alias` | 仅 developers；NOCASE |
| `color` | 仅 tags |
| `archived_at` | NULL=活跃；非 NULL=软删 |
| timestamps | |

**唯一索引（partial）：**

- `developers(alias) WHERE archived_at IS NULL`  
- `teams(name) WHERE archived_at IS NULL`  
- `tags(name) WHERE archived_at IS NULL`  

Web **默认不提供硬 DELETE**；归档设 `archived_at`。若强行 `DELETE` 且仍有 Activity → **RESTRICT 失败**。

### 6.3 关联表 + 反向索引

- `developer_teams(developer_id, team_id)`  
- `developer_tags(developer_id, tag_id)`  
- **索引：** `(team_id, developer_id)`、`(tag_id, developer_id)` — 避免按 Team/Tag 筛人时全表扫

### 6.4 `repos`

| 列 | 说明 |
|:---|:-----|
| `provider` | `ado` \| `github` |
| `org` / `project` / `name` | 展示与 az 参数（可变） |
| `external_id` | **稳定** repository GUID（ADO 必填推荐） |
| `enabled` | 是否参与采集 |
| `archived_at` | 软删 |
| `remote_url` | 可选 |

**索引：**

- UNIQUE `(provider, org, project, name) WHERE archived_at IS NULL`  
- UNIQUE `(provider, external_id) WHERE external_id IS NOT NULL AND archived_at IS NULL`

### 6.5 `activities`

| 列 | 说明 |
|:---|:-----|
| `developer_id` | FK **RESTRICT** |
| `type` | 开放字符串；一期约定见 01 |
| `occurred_at` | 事件 UTC 秒 |
| `day_key` | 当时 config 下的自然日 |
| `config_version` | 对应 `pipeline_config_version` |
| `provider` / `org` / `project` | 作用域（展示/过滤；幂等不依赖名称） |
| `repo_id` | PR 类有值；WI **NULL**；FK **RESTRICT** |
| `external_ref` | UNIQUE，§5 |
| `matched_unique_name` | 审计用邮箱串 |
| `meta_json` | 可选，`json_valid` |
| `ingested_at` | |

**索引：**

- UNIQUE `external_ref`  
- `(developer_id, day_key)`  
- `(developer_id, day_key DESC, occurred_at DESC)` — 明细时间线  
- `day_key`、`type`、`(org, project)`、`config_version`

### 6.6 `scores`

| 列 | 说明 |
|:---|:-----|
| `(developer_id, day_key)` | PK；developer FK **RESTRICT** |
| `config_version` | |
| `total` | 整型分 |
| `breakdown_json` | `json_valid` |
| `activity_count` | |
| `computed_at` | |

### 6.7 `unmatched_identities` / `ingest_runs`

同 0001；辅助未映射身份与运行审计。

## 7. 身份匹配（应用层）

与 01 一致，不因 §5 改变：

```
for suffix in email_suffixes:
  if lower(uniqueName) == lower(alias + "@" + suffix) → developer
```

- vote 的 `external_ref` 使用 **identity id/descriptor**；匹配 Developer 仍可用 uniqueName→alias。  
- bot / group：不建 Developer，不写 Activity。
