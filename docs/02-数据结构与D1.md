## 1. 目标

定义 signoff.now **一期**持久化结构，使：

1. Web 可 CRUD：Developer / Team / Tag / Repo / Settings  
2. 本机管线可幂等写入：Activity / Score（及辅助表）  
3. 字段 **provider 中立**（`ado` | `github`），一期只写 `ado`  
4. 完成阅读本文 + 执行 §8 命令后，**D1 库可完整建立**

实现文件：

| 路径 | 说明 |
|:-----|:-----|
| `packages/db/migrations/0001_initial.sql` | 初始 DDL + 默认 settings 种子 |
| `wrangler.toml` | D1 binding、`migrations_dir` |
| `packages/db/` | schema 包（占位 Worker entry 仅满足 wrangler） |

---

## 2. 设计原则

| 原则 | 决策 |
|:-----|:-----|
| 单实例 | 无 tenant_id；一套表即一份部署 |
| 主键 | `TEXT` UUID（应用/管线生成，非 SQLite AUTOINCREMENT 业务键） |
| 时间 | `INTEGER` unix **秒**（UTC）；业务「自然日」另存 `day_key` |
| 自然日 | `day_key TEXT` 格式 `YYYY-MM-DD`，按 **Settings.timezone** 切割（默认 `Asia/Shanghai`） |
| 大小写 | `developers.alias`、`teams.name`、`tags.name` 使用 `COLLATE NOCASE` + UNIQUE |
| 匹配 | `alias@email_suffix` 与 ADO `uniqueName` 忽略大小写全等（逻辑在应用层；库只存 alias） |
| WI 作用域 | Work Item **不挂 repo**；Activity 上 `repo_id` 可空，用 `org+project+external_ref` 表达 |
| 幂等 | `activities.external_ref` **全局 UNIQUE**；ingest 使用 UPSERT |
| 读写边界 | Activity / Score 仅管线写；Web 只读这两类 |
| 扩展 | 新贡献源 = 新 `activities.type` 字符串 + settings 权重键，不改表结构 |

## 4. 表定义

### 4.1 `settings`

键值表，`value` 为 **JSON 文本**。

| 列 | 类型 | 说明 |
|:---|:-----|:-----|
| `key` | TEXT PK | 配置键 |
| `value` | TEXT NOT NULL | JSON 序列化 |
| `updated_at` | INTEGER | unix 秒 |

**一期约定键：**

| key | value 形状 | 默认（migration 已种子） |
|:----|:-----------|:-------------------------|
| `timezone` | string | `"Asia/Shanghai"` |
| `email_suffixes` | string[] | `["microsoft.com"]` |
| `activity_weights` | Record\<type, number\> | 见 01 §6.3 |

应用读取时 `JSON.parse(value)`；更新时整键替换。

---

### 4.2 `developers`

| 列 | 类型 | 说明 |
|:---|:-----|:-----|
| `id` | TEXT PK | UUID |
| `name` | TEXT | 显示名 |
| `alias` | TEXT UNIQUE NOCASE | 短标识；匹配时拼 `alias@suffix` |
| `created_at` / `updated_at` | INTEGER | |

**匹配（应用层，非 SQL）：**

```
for suffix in settings.email_suffixes:
  if lower(uniqueName) == lower(f"{alias}@{suffix}"):
    → developer
```

未命中可写入 `unmatched_identities`（可选）。

---

### 4.3 `teams` / `tags` / 关联表

**teams：** `id`, `name` (UNIQUE NOCASE), timestamps  

**tags：** `id`, `name` (UNIQUE NOCASE), `color` (如 `#3B82F6`), timestamps  

**developer_teams / developer_tags：** 复合主键 `(developer_id, team_id|tag_id)`，`ON DELETE CASCADE`

一人多 Team、多 Tag。

---

### 4.4 `repos`

单 Git 仓库绑定。

| 列 | 类型 | 说明 |
|:---|:-----|:-----|
| `id` | TEXT PK | |
| `provider` | TEXT | `'ado'` \| `'github'` |
| `org` | TEXT | ADO organization 名 |
| `project` | TEXT | ADO project 名 |
| `name` | TEXT | 仓库名 |
| `remote_url` | TEXT NULL | 可选 |
| `external_id` | TEXT NULL | ADO repository GUID 等 |
| `enabled` | INTEGER 0/1 | 是否参与采集 |
| timestamps | | |

**唯一约束：** `(provider, org, project, name)`

采集清单：`SELECT * FROM repos WHERE enabled = 1`。

同一 `org+project` 下多个 repo 共享 **一份** WI 采集（见 Activity / 落盘约定）。

---

### 4.5 `activities`

管线写入的可计分事件；**可从 raw JSON 确定性重建**。

| 列 | 类型 | 说明 |
|:---|:-----|:-----|
| `id` | TEXT PK | |
| `developer_id` | TEXT FK → developers | |
| `type` | TEXT | 见下表 |
| `occurred_at` | INTEGER | 事件 UTC 秒（稳定源字段） |
| `day_key` | TEXT | 实例时区自然日 `YYYY-MM-DD` |
| `provider` | TEXT | 默认 `ado` |
| `org` / `project` | TEXT | 作用域坐标 |
| `repo_id` | TEXT NULL FK → repos | **PR 类必填逻辑上应有；WI 类为 NULL** |
| `external_ref` | TEXT UNIQUE | 幂等键（见 §5） |
| `matched_unique_name` | TEXT NULL | 命中时的源 uniqueName |
| `meta_json` | TEXT NULL | 展示用附属 JSON |
| `ingested_at` | INTEGER | 入库时间 |

**索引：** `(developer_id, day_key)`、`day_key`、`type`、`(org, project)`

#### 一期 `type` 枚举（开放字符串，下列为约定值）

| type | repo_id | 说明 |
|:-----|:--------|:-----|
| `pr.merged` | 有 | 作者，合并完成日 |
| `pr.closed` | 有 | 作者，关闭未合并 |
| `pr.created` | 有 | 作者，创建日 |
| `pr.vote` | 有 | **投票者**，需 threads 时间 |
| `pr.active` | 有 | 可选；需 iterations.updatedDate |
| `wi.created` | NULL | WI 创建 |
| `wi.updated` | NULL | WI 修订日（同 WI 同日可去重进 ref） |
| `wi.closed` | NULL | WI 关闭 |

权重不存行内，读 `settings.activity_weights` 计算 Score。

---

### 4.6 `scores`

物化日积分，供热力图/对比快读。

| 列 | 类型 | 说明 |
|:---|:-----|:-----|
| `developer_id` | TEXT | PK 之一 |
| `day_key` | TEXT | PK 之一 |
| `total` | INTEGER | 当日加权总分 |
| `breakdown_json` | TEXT | 分 type 小计 JSON |
| `activity_count` | INTEGER | 事件条数 |
| `computed_at` | INTEGER | 最近计算时间 |

**主键：** `(developer_id, day_key)`

管线在写入/删除 Activity 后按 `(developer_id, day_key)` 重算或增量更新（算法见 03）。

## 5. `external_ref` 约定（幂等）

格式建议（管线必须文档化并测试）：

| type | external_ref 示例 |
|:-----|:------------------|
| `pr.created` | `ado:pr:{org}/{project}/{repoName}/{prId}:created` |
| `pr.merged` | `ado:pr:{org}/{project}/{repoName}/{prId}:merged` |
| `pr.closed` | `ado:pr:{org}/{project}/{repoName}/{prId}:closed` |
| `pr.vote` | `ado:pr:{org}/{project}/{repoName}/{prId}:vote:{voterLocalOrEmail}:{threadOrCommentId}` |
| `pr.active` | `ado:pr:{org}/{project}/{repoName}/{prId}:active:{day_key}` 或带 iteration id |
| `wi.created` | `ado:wi:{org}/{project}/{wiId}:created` |
| `wi.closed` | `ado:wi:{org}/{project}/{wiId}:closed` |
| `wi.updated` | `ado:wi:{org}/{project}/{wiId}:updated:{day_key}` |

- `org` / `project` / `repoName` 使用绑定表中的规范字符串（与 `repos` 一致）。  
- WI **不包含 repo 段**，避免同 Project 多仓库重复插入。

## 6. 与落盘目录的对应（回顾 01）

```text
.data/raw/ado/{org}/{project}/
  repos/{repo}/prs|pr-threads|pr-iterations/…
  workitems/…              → activities (repo_id NULL)
  workitem-updates/…
```

ingest：raw → 校验 schema → 映射 developer → UPSERT `activities` → 重算 `scores`。

## 8. 建立 / 迁移 D1

### 8.1 前置

- `wrangler` 已登录（`npx wrangler whoami`）
- 仓库根目录 `wrangler.toml` 已配置 `database_id`

当前绑定（创建时写入；以 `wrangler.toml` 为准）：

| 项 | 值 |
|:---|:---|
| database_name | `signoff-db` |
| binding | `DB` |
| migrations_dir | `packages/db/migrations` |

### 8.2 命令

```bash
# 本地（Miniflare SQLite）
bun run db:migrate:local

# 远端
bun run db:migrate:remote

# 查看远端 migration 状态
bun run db:list:remote
```

等价：

```bash
npx wrangler d1 migrations apply signoff-db --local
npx wrangler d1 migrations apply signoff-db --remote
```

### 8.3 新建库（若从零）

```bash
npx wrangler d1 create signoff-db
# 将输出的 database_id 写入 wrangler.toml
bun run db:migrate:remote
```

### 8.4 冒烟查询

```bash
npx wrangler d1 execute signoff-db --remote --command \
  "SELECT key FROM settings ORDER BY key; SELECT name FROM sqlite_master WHERE type='table' ORDER BY name;"
```

期望：`settings` 含 `timezone` / `email_suffixes` / `activity_weights`；表集合含本文 §4 所列。

## 9. 应用层类型草图（TypeScript）

非 ORM 强制；供 Model 层对齐：

```ts
type Provider = "ado" | "github";

type ActivityType =
  | "pr.merged"
  | "pr.closed"
  | "pr.created"
  | "pr.vote"
  | "pr.active"
  | "wi.created"
  | "wi.updated"
  | "wi.closed"
  | (string & {}); // future

interface Developer {
  id: string;
  name: string;
  alias: string; // canonical lower or NOCASE
  createdAt: number;
  updatedAt: number;
}

interface Repo {
  id: string;
  provider: Provider;
  org: string;
  project: string;
  name: string;
  remoteUrl: string | null;
  externalId: string | null;
  enabled: boolean;
}

interface Activity {
  id: string;
  developerId: string;
  type: ActivityType;
  occurredAt: number;
  dayKey: string; // YYYY-MM-DD
  provider: Provider;
  org: string;
  project: string;
  repoId: string | null;
  externalRef: string;
  matchedUniqueName: string | null;
  metaJson: string | null;
}

interface Score {
  developerId: string;
  dayKey: string;
  total: number;
  breakdown: Record<string, number>;
  activityCount: number;
  computedAt: number;
}

interface AppSettings {
  timezone: string;
  emailSuffixes: string[];
  activityWeights: Record<string, number>;
}
```

列名 snake_case（SQL）↔ camelCase（TS）在 Model/Mapper 完成。

## 10. 明确不做（本 schema 阶段）

- 多租户 / tenant 列  
- GitHub 特有表（沿用同一 `repos`/`activities`，二期再填 `provider='github'`）  
- 在 D1 存 raw PR JSON 大对象（raw 在本机 `.data/`）  
- Web 可写 Activity/Score  
- Livesite / IM 专用表（仅预留 type 字符串空间）
